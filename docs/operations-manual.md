# 해군 사상자 관리 시스템 -- 운영자 매뉴얼

버전: 1.0.0 | 작성일: 2026-04-04

---

## 1. 시스템 개요

### 1.1 목적

해군 내 사망자, 상이자, 전공사상심사 대상자의 정보를 체계적으로 등록/관리하고, 공식 문서를 출력하며, 통계 현황을 산출하는 군 내부망 전용 전산 시스템입니다.

### 1.2 주요 기능

- **인원 관리**: 사망자/상이자/전공사상심사 대상자 등록, 조회, 수정, 삭제
- **문서 출력**: 국가유공자 확인서, 순직확인서, 현황 보고서 등 7종 공식 문서 PDF 출력
- **통계/현황**: 신분별, 월별, 연도별, 부대별 통계 차트 및 Excel 다운로드
- **감사 로그**: 모든 활동(조회, 등록, 수정, 삭제, 출력) 이력 자동 기록
- **관리자 기능**: 사용자 관리, 데이터 임포트, 감사 보고서

### 1.3 접속 방법

- 접속 URL: `https://<서버IP>:8443`
- 지원 브라우저: Chrome, Edge (최신 버전 권장)
- 군 내부망 PC에서만 접속 가능

## 2. 역할 및 권한

시스템은 역할 기반 접근 제어(RBAC)를 적용합니다. 상위 역할은 하위 역할의 모든 권한을 포함합니다.

| 역할 | 조회 | 등록/수정 | 삭제 | 문서 출력 | 관리자 기능 | Excel 다운로드 |
|------|------|----------|------|---------|------------|---------------|
| VIEWER | O | X | X | X | X | X |
| OPERATOR | O | O | X | O | X | O |
| MANAGER | O | O | O | O | X | O |
| ADMIN | O | O | O | O | O | O |

### 역할 계층

```
ADMIN > MANAGER > OPERATOR > VIEWER
```

- **VIEWER**: 데이터 조회만 가능 (신규 인원 열람용)
- **OPERATOR**: 데이터 등록/수정 + 문서 출력 (일반 담당자)
- **MANAGER**: 데이터 삭제 권한 추가 (선임 담당자)
- **ADMIN**: 사용자 관리, 시스템 설정, 감사 로그 조회 (시스템 관리자)

## 3. 로그인/로그아웃

### 3.1 로그인

1. 접속 URL 입력 후 로그인 화면 표시
2. 사용자명과 비밀번호 입력
3. [로그인] 버튼 클릭

### 3.2 초기 비밀번호 변경

- 관리자가 제공한 초기 비밀번호로 첫 로그인 시 **비밀번호 변경 페이지로 자동 이동**
- 비밀번호 변경을 완료해야 시스템 사용 가능

### 3.3 계정 잠금

- 비밀번호 5회 연속 오류 시 계정 자동 잠금
- 잠금 해제: 관리자(ADMIN)에게 요청

### 3.4 자동 로그아웃

- 30분 무조작 시 세션 자동 만료
- 재로그인 필요

### 3.5 로그아웃

- 우측 상단 사용자 아이콘 클릭 > [로그아웃] 선택

## 4. 사망자 관리

좌측 메뉴에서 **"사망자 현황"** 클릭하여 접근합니다.

### 4.1 목록 조회

- 검색 조건: 군구분, 군번, 성명, 생년월일, 계급, 소속
- [조회] 버튼 클릭하여 검색 실행
- 서버사이드 페이징 적용 (페이지당 10/20/50건 선택 가능)
- 주민등록번호는 역할에 따라 마스킹 표시 (예: 850101-1\*\*\*\*\*\*)

### 4.2 사망자 등록

1. [등록] 버튼 클릭 > 등록 Modal 팝업
2. 필수 입력 항목:

| 항목 | 설명 | 비고 |
|------|------|------|
| 군번 | 군인 고유번호 | 중복 자동 차단 |
| 성명 | 한글 이름 | |
| 주민등록번호 | 13자리 | 형식 자동 검증 |
| 계급 | 드롭다운 선택 | |
| 소속 | 부대명 | |
| 사망구분 | 전사/순직/사고사 등 | |

3. [저장] 버튼 클릭
4. 주민번호 7번째 자리로 생년월일 자동 추출

### 4.3 사망자 수정

1. 목록에서 수정할 행 클릭
2. 수정 Modal에서 데이터 변경
3. [저장] 버튼 클릭

### 4.4 사망자 삭제

- **MANAGER 이상 권한** 필요
- 삭제 시 삭제 사유 필수 입력
- 논리 삭제 처리 (데이터 보존, 감사 이력 유지)

### 4.5 Excel 다운로드

- 현재 검색 결과 기준으로 Excel 파일 다운로드
- 주민등록번호 마스킹 적용 상태로 내보내기
- 최대 10,000행 제한 (대용량 데이터 시 검색 조건 세분화)

## 5. 상이자 관리

좌측 메뉴에서 **"상이자 현황"** 클릭하여 접근합니다.

사망자 관리와 동일한 CRUD 구조이며, 다음 고유 필드가 추가됩니다:

| 항목 | 설명 |
|------|------|
| 보훈청명 | 관할 보훈청 |
| 병명 | 상이 관련 병명/진단명 |
| 상이구분 | 전상/공상 등 상이 분류 |

### 상이자 상태 관리

상이자는 4단계 상태 전이를 거칩니다:

```
등록(REGISTERED) -> 심사중(UNDER_REVIEW) -> 확정(CONFIRMED) -> 통보완료(NOTIFIED)
```

## 6. 전공사상심사 관리

좌측 메뉴에서 **"전공사상심사"** 클릭하여 접근합니다.

### 6.1 심사 등록/수정

- 심사차수, 심사일자, 성명, 군번, 주민번호, 입대일자, 소속, 병명, 소속부대 심사결과, 전공상 분류 입력
- 동일인에 대해 복수 심사 등록 가능 (심사차수별 관리)
- 군번 + 심사차수 중복 자동 차단

### 6.2 심사 결과 반영

- 심사 결과 저장 시 연계된 사망자/상이자 분류가 자동 반영
- 분류 변경:
  - 사망자: 사망구분(deathType) 자동 업데이트
  - 상이자: 상이구분(woundType) 자동 업데이트

### 6.3 심사 이력 조회

- 대상자별 심사차수 이력을 타임라인으로 시각화
- Drawer 패널에서 상세 이력 확인 가능

### 6.4 보훈청 통보

- 보훈청 통보 버튼 클릭 시 확인 Modal 표시
- 통보일자 자동 기록
- **4단계 상태 전이 완료 후에만 통보 가능**

## 7. 문서 출력

좌측 메뉴에서 **"문서 출력"** 을 통해 접근합니다.

### 7.1 출력 가능 문서 (7종)

| 번호 | 문서명 | 대상 |
|------|--------|------|
| 1 | 국가유공자 요건 해당사실 확인서 (사망자용) | 사망자 |
| 2 | 국가유공자 요건 해당사실 확인서 (상이자용) | 상이자 |
| 3 | 전공사상심사결과서 | 전공사상심사 대상자 |
| 4 | 순직/사망확인서 | 사망자 |
| 5 | 사망자 현황 보고서 | 사망자 전체 |
| 6 | 상이자 현황 보고서 | 상이자 전체 |
| 7 | 전사망자 확인증 발급대장 | 사망자 전체 |

### 7.2 출력 절차

1. 대상자 목록에서 문서 출력 버튼 클릭
2. 문서 유형 선택
3. **발급 목적 입력** (필수 -- 감사 추적용)
4. PDF 미리보기 표시
5. [인쇄] 또는 [다운로드] 클릭

### 7.3 발급 이력

- 모든 문서 발급은 자동으로 이력 기록 (발급자, 발급일시, 발급목적, 대상자)
- 문서 발급 이력 페이지에서 조회 가능

좌측 메뉴에서 **"통계/현황"** 하위 메뉴로 접근합니다.

| 부대별 현황 | 부대별 사망자 분포 | 막대 차트 |

### 8.2 명부

## 9. 관리자 기능

좌측 메뉴에서 **"관리"** 클릭하여 접근합니다. **ADMIN 역할만 접근 가능합니다.**

### 9.1 사용자 관리

- 신규 사용자 계정 생성
- 역할 부여/변경 (VIEWER, OPERATOR, MANAGER, ADMIN)
- 계정 잠금 해제 (비밀번호 5회 오류로 잠긴 계정)
- 비밀번호 초기화

### 9.2 감사 로그 조회

- 시스템의 모든 활동 이력 조회
- 검색 조건: 수행자, 활동 유형, 대상, 기간
- 기록 항목: 누가(사용자), 언제(일시), 어떤 작업(활동 유형), 대상(엔티티/ID)

### 9.3 데이터 임포트

- Excel 파일을 통한 대량 데이터 일괄 등록
- 지원 유형: 사망자, 상이자, 전공사상심사
- 업로드 후 자동 검증 (형식, 필수값, 중복 체크)
- 오류 행 표시 및 수정 안내

### 9.4 감사 보고서

- 월별 감사 활동 요약 PDF 생성
- 활동 유형별 건수 통계
- 이상 활동 감지 (대량 조회, 비정상 시간대 접근 등)

### 9.5 데이터 정합성 확인

- `/api/admin/data-check` 엔드포인트로 코드 테이블 정합성 검증
- 군구분, 계급, 부대코드 등 참조 데이터 무결성 확인

## 10. 업무 시나리오 가이드

### 시나리오 1: 사망자 등록 후 확인서 발급

1. 좌측 메뉴 **"사망자 현황"** 클릭
2. [등록] 버튼 클릭 > 사망자 정보 입력 > [저장]
3. 등록된 사망자 행에서 **문서 출력** 버튼 클릭
4. **"국가유공자 요건 해당사실 확인서(사망자용)"** 선택
5. 발급 목적 입력 (예: "유족 제출용")
6. PDF 미리보기 확인 > [인쇄] 또는 [다운로드]

### 시나리오 2: 전공사상심사 결과 처리

1. 좌측 메뉴 **"전공사상심사"** 클릭
2. [등록] 버튼 클릭 > 심사 정보 입력 (심사차수, 분류 결과 등)
3. [저장] > 사망자/상이자 분류 자동 반영 확인
4. 보훈청 통보 필요 시: **[보훈청 통보]** 버튼 클릭
5. 확인 Modal에서 [확인] > 통보일 자동 기록

### 시나리오 3: 기존 데이터 일괄 등록 (Excel 임포트)

1. 좌측 메뉴 **"관리"** 클릭 (ADMIN 역할 필요)
2. **"데이터 임포트"** 탭 선택
3. 유형 선택 (사망자 / 상이자 / 전공사상심사)
4. Excel 파일 업로드 > 자동 검증 실행
5. 검증 결과 확인:
   - 성공: 등록 건수 표시
   - 오류: 오류 행 번호 및 사유 표시 > 수정 후 재업로드

### 시나리오 4: 월별 통계 확인 및 보고서 작성

1. 좌측 메뉴 **"통계/현황"** > **"월별 현황"** 클릭
2. 연도 선택 후 차트/표 확인
3. [Excel 다운로드] 클릭하여 원본 데이터 확보
4. 필요 시 다른 통계 메뉴에서 추가 데이터 수집

## 11. 에러 대응 가이드

| 증상 | 원인 | 대응 |
|------|------|------|
| "네트워크 오류" | 서버 접속 불가 또는 서버 장애 | 서버 상태 확인 (관리자에게 문의) |
| "권한이 없습니다" | 현재 역할에 해당 기능 권한 없음 | 관리자에게 역할 변경 요청 |
| "이미 등록된 군번" | 동일 군번 중복 등록 시도 | 기존 데이터 확인 후 수정으로 처리 |
| "세션이 만료되었습니다" | 30분 무조작 후 자동 로그아웃 | 재로그인 |
| "주민번호 형식 오류" | 잘못된 주민번호 입력 | 13자리 숫자 형식 확인 (XXXXXX-XXXXXXX) |
| "필수 입력값 누락" | 필수 항목 미입력 | 빨간색 표시된 필수 항목 확인 |
| "삭제 사유를 입력하세요" | 삭제 시 사유 미입력 | 삭제 사유 입력 후 재시도 |
| "계정이 잠겼습니다" | 비밀번호 5회 연속 오류 | 관리자에게 잠금 해제 요청 |
| "파일 형식 오류" | Excel 임포트 시 잘못된 파일 | .xlsx 형식 확인, 템플릿 양식 준수 |

## 부록 B: 용어 정리

| 용어 | 설명 |
|------|------|
| 논리 삭제 | 데이터를 실제 삭제하지 않고 삭제 표시만 하는 방식 |
| RBAC | 역할 기반 접근 제어 (Role-Based Access Control) |
| 감사 로그 | 시스템 내 모든 활동 이력 자동 기록 |
| 전공사상 | 전투/공무 중 발생한 사상 (사망 또는 부상) |
| PII | 개인식별정보 (Personally Identifiable Information) |
