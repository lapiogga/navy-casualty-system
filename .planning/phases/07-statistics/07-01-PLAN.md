    - JPAQueryFactory 주입 (@RequiredArgsConstructor)
    - @Repository 어노테이션
    - getByBranch(): `from(dead).leftJoin(branch).on(dead.branchId.eq(branch.id)).where(dead.deletedAt.isNull()).groupBy(branch.id, branch.branchName)` + Projections.constructor
    - getByMonth(): `Expressions.numberTemplate(Integer.class, "EXTRACT(MONTH FROM {0})", dead.deathDate)` + EXTRACT(YEAR) 결합, where(dead.deletedAt.isNull()), groupBy(yearExpr, monthExpr), orderBy(yearExpr.desc(), monthExpr.desc())
    - getByYear(): EXTRACT(YEAR FROM dead.deathDate)만 사용, 나머지 동일
    - getByUnit(): getByBranch()와 동일 패턴, UnitCode JOIN
    - getRosterByUnit(unitId): `selectFrom(dead).where(dead.deletedAt.isNull().and(dead.unitId.eq(unitId))).orderBy(dead.id.desc()).fetch()`
    - getRosterAll(): `selectFrom(dead).where(dead.deletedAt.isNull()).orderBy(dead.id.desc()).fetch()`
    - 중요: 모든 쿼리에 `.where(dead.deletedAt.isNull())` 명시 (Projections 사용 시 @SQLRestriction 미적용 방지)
    - NULL branch_id/unit_id 처리: leftJoin 사용 + COALESCE로 branchName/unitName이 null이면 "미분류" 반환
