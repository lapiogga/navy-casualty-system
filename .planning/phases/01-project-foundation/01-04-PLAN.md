<task type="auto">
  <name>Task 2: App.tsx dead code 삭제 및 client.ts 401 리다이렉트를 router.navigate로 전환</name>
  <files>frontend/src/App.tsx, frontend/src/api/client.ts</files>
  <action>
UAT Gap 2의 실제 root cause: client.ts의 `window.location.href = '/login'`이 full page reload를 일으켜 React Router 상태(AuthProvider 등)가 초기화되면서 리다이렉트가 정상 작동하지 않음.

**수정 1 - App.tsx 삭제:**
- App.tsx는 main.tsx에서 import되지 않는 dead code이다 (main.tsx는 직접 routes/index.tsx의 router를 RouterProvider에 전달).
- 프로젝트 어디에서도 App.tsx를 import하지 않으므로 파일을 삭제한다.
- `rm frontend/src/App.tsx`

**수정 2 - client.ts 401 인터셉터를 router.navigate로 전환:**
- 현재: `window.location.href = '/login'` (full page reload → React state 초기화)
- 변경: `router.navigate('/login')` (SPA 내 네비게이션 → state 유지)
- routes/index.tsx에서 export되는 router 인스턴스를 import하여 사용:

```typescript
import axios from 'axios';
import router from '../routes';

const apiClient = axios.create({
  baseURL: '/api',
  timeout: 30000,
  headers: { 'Content-Type': 'application/json' },
});

// 세션 만료 시 로그인 페이지로 리다이렉트 (/login 경로에서는 무한 루프 방지)
apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      if (window.location.pathname !== '/login') {
        router.navigate('/login');
      }
    }
    return Promise.reject(error);
  }
);

export default apiClient;
```

**주의:** window.location.href 방식을 router.navigate로 바꾸는 것이 핵심. full page reload 대신 React Router의 SPA 네비게이션을 사용해야 AuthProvider 상태가 유지된다.

수정 후 `npm run build`로 빌드 성공 확인.
